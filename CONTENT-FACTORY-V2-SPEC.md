# BRITZMEDI 콘텐츠 팩토리 v2 — 전체 설계 문서

> 이 문서는 콘텐츠 팩토리의 전면 재설계 스펙이다.
> 기존 콘텐츠 팩토리(자유 텍스트 → AI 생성)를 대체한다.
> 구현 순서대로 Phase를 따라가면 된다.

---

## 0. 핵심 원칙

### 최종 비전
```
📱 외부에서 모바일로 메시지 몇 자 → Content Ops가 알아서 제작 → 사용자는 컨펌만
```

### 설계 철학
1. **입력은 대충, 처리는 정확하게** — 사용자는 자유 텍스트로 대충 던진다. AI가 구조화한다.
2. **할루시네이션 구조적 차단** — "지어내지 마"라는 프롬프트가 아니라, 입력 안 한 건 못 쓰게 구조로 막는다.
3. **팩트 확인 관문** — AI 파싱 결과를 사용자가 확인하는 단계가 반드시 있다.
4. **퀄리티 = 자동 검수** — 생성 후 AI가 자동 검수하고, 통과해야 내보내기 가능.
5. **최소 터치** — 사용자 액션은 "대충 입력 → 확인 → 컨펌" 3번 이내.

## 1. 전체 플로우 아키텍처

```
📱 STEP 0: 자유 텍스트 입력
   "학회 참가. 토르RF 루미노웨이브. 이상수 원장 라이브"
        │
        ▼
🤖 STEP 1: AI 파싱 ──── (1차 API 호출)
   - 카테고리 자동 판별
   - 자유 텍스트 → 구조화 필드 추출
   - 소스에 없는 건 "(소스에 없음)" 표시
   - 절대 추측/상상으로 채우지 않음
        │
        ▼
👆 STEP 2: 사용자 확인 ──── (터치 한 번)
   - 파싱 결과를 카드 형태로 보여줌
   - 각 필드 편집 가능 (빈 필드 채우기, 틀린 거 수정)
   - "확인 완료 → 생성" 버튼
        │
        ▼
🤖 STEP 3: 콘텐츠 생성 ──── (2차 API 호출)
   - 확인된 팩트만 사용
   - 빈 필드 → [입력 필요] 플레이스홀더
   - 채널별 포맷 규칙 적용
   - AI 역할 = "팩트의 문장화 + 포맷 편집", 창작 아님
        │
        ▼
🤖 STEP 4: 자동 검수 ──── (3차 API 호출)
   - STEP 2에서 확인된 팩트 vs 생성 결과 문장 대조
   - 의료법 금지어, 표기법, 구조 체크
   - 팩트 비율 계산 (50% 미만이면 🔴)
   - 결과: 🔴 n건 / 🟡 n건 / ✅ 통과
        │
        ▼
👆 STEP 5: 최종 컨펌 ──── (터치 한 번)
   - ✅ 통과 → "컨펌" → 내보내기 + 파이프라인 등록
   - 🔴 있음 → 해당 부분만 수정 후 컨펌
```

## 3. 보도자료 카테고리별 파싱 필드

AI가 자유 텍스트를 받으면 카테고리를 자동 판별한 뒤, 해당 카테고리의 필드로 파싱한다.
사용자가 카테고리를 직접 변경할 수도 있다.

### 3-1. 전시회·학회 참가

| 필드명 | 필수 | 설명 |
|-------|-----|------|
| 행사명 | ● | 학회/전시회 공식 명칭 |
| 일시 | ● | 날짜 또는 기간 |
| 장소 | ○ | 개최지 |
| 전시 제품 | ● | 제품 목록 + 각 제품 역할(주력/신제품 등) |

| 미팅/비즈니스 성과 | ○ | 구체적 건수, 논의 내용 (팩트만) |

### 4-2. 에디터 핵심 규칙

1. **라벨은 셀 밖** — 셀 안에는 순수 텍스트만. 긁었을 때 깔끔하게 복사됨.
2. **개별 편집** — 각 셀 클릭하면 편집 모드. 셀 단위로 수정 가능.
3. **전체 복사 한 번에** — "전체 복사" 버튼 = 모든 셀 내용을 뉴스와이어 순서대로 합침. 라벨 없이 순수 텍스트만. 항목 사이 줄바꿈 2번으로 구분.
4. **[입력 필요] 플레이스홀더** — 빈 필드에서 나온 부분은 노란색 하이라이트. 편집하지 않으면 전체 복사에서 제외.
5. **검수 🔴가 있으면** — Word/PDF 다운로드, 파이프라인 등록 버튼 비활성. "수정 후 내보내기" 안내 표시.

### 4-3. 내보내기

#### Word (.docx) 다운로드
- 상단: "[보도자료]" 표기 + BRITZMEDI 로고 영역(placeholder)
- 제목: 굵게, 16pt
- 부제목: 14pt
- 본문: 11pt, 줄간격 1.6
- 하단: 회사 소개 + 연락처
- A4, 깔끔한 비즈니스 레이아웃

#### PDF 다운로드
- Word와 동일 레이아웃을 PDF로 변환

#### 뉴스와이어 전체 복사
- 클립보드에 전체 텍스트 (라벨 없이) 복사

### 4-4. 보도자료 → 채널 콘텐츠 재가공

파이프라인에 "발행완료"로 등록된 보도자료 카드에 "채널 콘텐츠 만들기" 버튼이 있다.

**플로우:**
1. "채널 콘텐츠 만들기" 클릭
2. 채널 선택 (뉴스레터/네이버/카카오톡/LinkedIn/Instagram 복수선택)
3. AI가 보도자료 원문을 각 채널 포맷에 맞게 재가공 (보도자료 전문이 AI 소스로 자동 입력됨)
4. 보도자료 원문 = 검증된 팩트이므로 파싱 확인 단계(STEP 2) 생략 가능
5. 채널별 결과 + 자동 검수
6. 생성된 각 채널 콘텐츠 → 파이프라인 "초안작성"으로 등록

```
당신은 BRITZMEDI 콘텐츠 품질 검수 AI입니다.

## 검수 대상

- "생성된 콘텐츠": AI가 작성한 결과물 (이것을 검수)

## 검수 규칙 (항목별)

### [FACT] 사실 검증 — 가장 중요
- 생성된 콘텐츠의 모든 문장을 "확인된 팩트"와 하나씩 대조한다.

- 구체적 숫자(참석자 수, 미팅 건수 등)가 확인된 팩트에 없으면 🔴
- AI가 만든 인용문이 있으면 🔴 "확인되지 않은 인용문"
- 실존 여부 확인 필요한 행사명, 기관명, 인물명 → 🟡 "사실 확인 필요: [항목]"
- 팩트 비율 계산: (팩트 기반 문장 수 / 전체 문장 수) × 100%. 50% 미만이면 🔴 경고.

### [TIME] 시제/시점
- 예고형인데 과거 시제 → 🔴 "시제 불일치"
- 리뷰형인데 미래 시제(향후 계획 제외) → 🟡 "시제 확인"
- 리뷰형인데 소스에 없는 현장 반응/결과 → 🔴 "팩트 없는 리뷰 내용"

### [LAW] 의료법 준수
- 금지어 목록: 극대화, 최소화, 최고, 최초(근거 없이), 완벽, 획기적, 혁명적, 완치, 100%, 확실한 효과, 부작용 없음, 안전 보장
- 위 단어 사용 → 🔴 "의료법 위반 표현: [단어]" + 대체 표현 제안
- 효능 단정 ("~를 치료합니다", "~가 사라집니다") → 🔴 "효능 단정"
- 비교 광고 (타사 제품명 언급) → 🔴 "비교 광고"

### [NOTATION] 표기법
- 영문 약어: 첫 등장 시 "국문(영문)" 형태인지 확인 → 위반 시 🟡 "영문 표기 규칙 위반: [단어]"
  - 예: "FDA" 첫 등장 → "미국 식품의약국(FDA)" 이어야 함
  - 예: "EBD" 첫 등장 → "에너지 기반 디바이스(EBD)" 이어야 함
  - 예: "TOROIDAL" 첫 등장 → "토로이달(TOROIDAL)" 이어야 함
  - 이후 등장에서는 국문만 사용
- 회사명, 제품명 표기 일관성 확인

### [STRUCT] 구조 검증
- 보도자료: 보일러플레이트 포함 여부 → 누락 시 🔴
- 보도자료: 연락처 포함 여부 → 누락 시 🔴
- 보도자료: 발행 날짜 포함 여부 → 누락 시 🟡
- 제목/리드문에 언급한 제품이 본문에 설명 있는지 → 없으면 🔴 "언급만 하고 설명 없음: [제품명]"
- 뉴스레터: CTA 포함 여부 → 누락 시 🟡
- 네이버: 태그 10개 포함 여부 → 누락 시 🟡

### [DATA] 데이터 관련성

- 한 문장 60자 초과 → 🟡 "장문"

## 9. 구현 순서

### Phase A: 보도자료 팩토리 재구축 (최우선)
1. STEP 0 UI — 자유 텍스트 + 시점 선택
2. STEP 1 파싱 프롬프트 구현 + 카테고리별 필드 파싱
3. STEP 2 파싱 결과 확인 UI (필드별 카드, 편집 가능)
4. STEP 3 생성 프롬프트 구현 (팩트만 사용, 플레이스홀더)
5. STEP 4 검수 프롬프트 구현 (JSON 응답 파싱, UI 표시)
6. 보도자료 에디터 UI (셀 분리, 전체 복사, Word/PDF 내보내기)
7. 파이프라인 "발행완료" 등록

### Phase B: 채널 재가공
8. 보도자료 → 채널 콘텐츠 재가공 플로우
9. 채널별 포맷 규칙 프롬프트
10. 각 채널 결과 + 검수

### Phase C: 일반 콘텐츠 팩토리 적용
11. 보도자료 외 채널(뉴스레터, 네이버 등) 직접 생성에도 파싱→확인→생성→검수 플로우 적용
12. 필라별 프리셋 주제 통합

### Phase D: API 프록시 + DB
13. Cloudflare Workers API 프록시 (API Key 입력 제거)
14. Supabase 연동 (콘텐츠 DB 저장)
15. 콘텐츠 블럭 시스템 (재사용 가능한 문단 관리)

## 10. ⚠️ 필수: 검수 테스트 (구현 완료 후 반드시 실행)

### 구현이 끝나면 아래 테스트를 모두 통과해야 한다. 하나라도 실패하면 수정 후 재테스트.

### 테스트 1: 할루시네이션 차단 — 짧은 소스

**입력:**
```
소스: "2026 춘계 대한피부미용레이저학회 참가. 토르RF 주력, 루미노웨이브 신제품 전시. 미라벨피부과 이상수 원장 토르RF 라이브세션 진행."
시점: 사후 배포(리뷰형)
```

**STEP 1 파싱 검증:**
- [ ] 행사명: "춘계 대한피부미용레이저학회" 또는 유사 (소스에서 추출)
- [ ] 일시: "2026년" (구체 날짜 = null 또는 "소스에 없음")
- [ ] 장소: null 또는 "소스에 없음" (소스에 장소 없으므로)
- [ ] 전시 제품: 토르RF(주력), 루미노웨이브(신제품)
- [ ] 라이브: 미라벨피부과 이상수 원장, 토르RF
- [ ] 미팅/성과: null (소스에 없으므로)
- [ ] 향후 계획: null (소스에 없으므로)
- [ ] 대표 인용문: null (소스에 없으므로)
- [ ] ❌ FAIL 조건: 위 null 필드에 AI가 값을 채워넣으면 실패

**STEP 3 생성 검증:**
- [ ] 본문에 소스에 없는 숫자(참석자 수, 미팅 건수 등)가 없어야 함
- [ ] 소스에 없는 현장 반응/질의응답 내용이 없어야 함
- [ ] 소스에 없는 대표 인용문이 없어야 함 → [대표 인용문 — 직접 작성 필요] 플레이스홀더
- [ ] 루미노웨이브에 대해 소스에 없는 상세 설명("사용성 개선" 등)이 없어야 함
- [ ] 소스에 없는 향후 계획("동남아 진출" 등)이 없어야 함
- [ ] 보일러플레이트 포함
- [ ] 연락처 포함
- [ ] ❌ FAIL 조건: 위 항목 중 하나라도 AI가 지어낸 내용이 있으면 실패

**STEP 4 검수 검증:**
- [ ] 만약 생성 결과에 소스 없는 내용이 있으면 검수가 🔴로 잡아야 함
- [ ] 팩트 비율 표시 (50% 미만이면 🔴)
- [ ] ❌ FAIL 조건: 소스 없는 내용이 있는데 검수가 통과(✅)시키면 실패

- [ ] 생성 단계에서 "획기적" → "우수한"/"높은 수준의" 등으로 자동 변환

### 테스트 6: 보도자료 에디터 기능

- [ ] 각 필드가 독립된 편집 가능한 셀로 표시
- [ ] 셀 안에 라벨([제목], [본문] 등) 텍스트가 포함되지 않음
- [ ] "전체 복사" 버튼 → 클립보드에 라벨 없이 순수 텍스트만 복사
- [ ] 항목 사이 줄바꿈 2번으로 구분
- [ ] [입력 필요] 플레이스홀더가 있는 부분은 전체 복사에서 제외
- [ ] Word 다운로드 → .docx 파일 생성, 보도자료 양식 레이아웃
- [ ] PDF 다운로드 → .pdf 파일 생성
- [ ] 🔴 검수 이슈가 있으면 Word/PDF/파이프라인 버튼 비활성

### 테스트 7: 보도자료 → 채널 재가공

- [ ] 버튼 클릭 → 채널 선택 화면

- [ ] 각 채널 콘텐츠 → 파이프라인 "초안작성"으로 등록
- [ ] 생성된 콘텐츠에 보도자료에 없는 새로운 팩트가 추가되지 않았는지 확인
