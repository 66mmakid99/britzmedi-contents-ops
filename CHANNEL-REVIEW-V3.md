"원장님께서 장비를 선택하실 때 가장 중요하게 보시는 것 중 하나가 바로 '검증된 기술력'이실 겁니다. 태국처럼 메디컬 에스테틱 시장이 성숙한 국가에서 20년 경력의 유통사가 독점 계약을 선택했다는 건, 그만큼 토로이달 고주파 기술의 임상 가치를 인정받았다는 의미입니다."

**② 소제목이 의미 해석형으로 전환됨**

- "3년간 900대 규모, 태국 유통사가 선택한 신뢰" ✅ (팩트+의미)

- "토로이달 고주파 기술, 어떤 점이 다를까요?" ✅ (질문형)

### 보강 필요 사항

**① 도입부 "오늘 읽어야 하는 이유" 여전히 약함**

현재: "원장님, 안녕하세요. 브릿츠메디입니다. 오늘은 국내 개발 장비가 해외 시장에서 어떤 평가를 받고 있는지, 원장님 클리닉의 장비 선택에 참고가 될 소식을 전해드립니다."

→ 방향은 맞지만 "원장님 클리닉의 장비 선택에 참고가 될 소식"이 너무 간접적. "왜 지금 이걸 읽어야 하는지" 한 줄의 훅이 더 필요.

```
개선 방향:
"원장님, 안녕하세요. 브릿츠메디입니다. 
태국 피부과 시장에서 20년 경력의 유통 전문기업이 
수많은 장비 중 토르RF를 독점 공급 파트너로 선택했습니다.
이 선택이 원장님 클리닉에 어떤 의미가 있는지 전해드립니다."
→ 팩트(20년 경력 유통사의 선택) → 궁금증 유발
```

**② "3년간 총 900대"라는 표현 — 팩트 확인 필요**

보도자료 원문: "3년간 연 300대 규모"
이메일: "3년간 총 900대 수준" (300 × 3 = 900)

→ 계산상 맞지만, 보도자료에 "총 900대"라는 표현이 없으므로 주의. 공식 발표에 없는 수치를 AI가 자의적으로 계산해서 쓰면 팩트 검증에서 문제될 수 있음.

```
프롬프트 보강:
"보도자료에 명시된 숫자만 사용하세요. 
AI가 자체적으로 곱하거나 합산하여 새로운 수치를 만들지 마세요.
예: 보도자료에 '3년간 연 300대'라고 되어 있으면 그대로 사용.
'총 900대'처럼 계산한 수치는 사용 금지."
```

**③ 부제목이 한글자도 수정 없이 보도자료에서 가져온 패턴**

제목: "토르RF, 태국 시장 3년간 900대 공급 계약 체결"
부제목: "국내 토로이달 고주파 기술, 태국 피부과 20년 경력 유통사가 선택한 이유"

→ 부제목은 좋으나, 제목에 "900대"가 또 나옴 (팩트 이슈와 연동). 제목을 "토르RF, 태국 피부과 시장 독점공급 계약 체결"로 간결하게 하고 부제목에 수치를 넣는 게 나음.

**점수 근거: 85점** — 원장님 관점 전환이 잘 되어 있고, 구조도 안정적. 하지만 도입부 훅 약함 + 팩트 자의 계산 문제로 5점 감점.

- "[IMAGE: 토르RF 제품 전면 이미지. 흰색 배경에 고급스러운 디자인...]"
→ 이미지 선정 가이드로 즉시 활용 가능

**④ 기술 설명에 비유 사용**
"토로이달 방식은 마치 샤워기 물줄기가 고르게 퍼지듯이, 에너지를 넓고 균일하게 전달해서 콜라겐 리모델링을 촉진하면서도 시술 후 회복 시간을 줄여준다고 해요."
→ 비전문가 독자가 이해하기 쉬운 비유. 블로그 톤에 적합.

### 심각한 문제 ❌

**① "제목" / "부제목" / "도입부" / "핵심요약" 라벨이 그대로 노출됨**

화면에 보이는 것:
```
SEO 키워드
브릿츠메디, 토르RF, 메디컬에스테틱디바이스...

제목
브릿츠메디 토르RF, 태국 독점유통 계약 체결 - 3년간 연 300대 규모

부제목
FDA 승인 토로이달 고주파 기술, 동남아 메디컬 에스테틱 시장 본격 진출

도입부
동남아시아 메디컬 에스테틱 시장이 빠르게 성장하고 있다는 건...
```

→ "제목", "부제목", "도입부", "핵심요약"이라는 **구조 라벨이 본문에 그대로 출력**되고 있음. 이대로 복사해서 네이버 블로그에 붙여넣으면 라벨까지 같이 들어감.

이건 프롬프트에서 AI에게 "구조를 표시하기 위해 라벨을 써라"고 지시했기 때문에 발생하는 문제입니다. AI가 생성할 때는 구조를 구분하기 위해 라벨이 필요하지만, 최종 출력에서는 제거해야 합니다.

```
해결 방법 (2가지 중 택1):

방법 A — 프롬프트 수정:
"구조 라벨(제목, 부제목, 도입부, 핵심요약 등)을 본문에 표시하지 마세요.
라벨 없이 바로 내용만 작성하세요.
제목은 첫 줄에, 부제목은 둘째 줄에, 도입부는 세 번째 단락부터 시작합니다."

방법 B — 코드에서 후처리:
channelGenerate.js 또는 RepurposeHub.jsx에서
생성된 텍스트의 "제목\n", "부제목\n", "도입부\n", "핵심요약\n" 등의
라벨 라인을 자동 제거하는 stripLabels 함수 추가
```

**② 글자수 1,953자 — 목표 2,000자 미달 (47자 부족)**

아주 근소한 차이지만, 빨간색으로 표시되고 있을 것임. "핵심요약" 섹션이 중간에 잘린 것으로 보이므로, 마무리 문장이 완성되면 2,000자를 넘길 가능성 있음.

```
프롬프트 보강:
"반드시 2,200자 이상 작성하세요. (여유 마진 포함)"
→ 2,000자 정확히 맞추려 하면 자주 미달되므로 상한을 올려서 지시
```

**③ 마무리가 잘림**

"의료기기 수출이나 메디컬 에스테틱 디바이스에 관심 있는 분들이라"
→ 문장이 끊겨 있음. 마무리 문단이 완성되지 않았음.

**④ SEO 키워드가 본문 상단에 나열됨**

"SEO 키워드" 라벨 아래에 키워드 목록이 본문 가장 위에 위치. 네이버 블로그에서 키워드를 상단에 나열하는 것은 네이버 검색 알고리즘에 불리할 수 있음 (키워드 스터핑으로 인식).

```
프롬프트 보강:
"SEO 키워드는 본문 최하단에 배치하세요.
본문 상단에 키워드 목록을 나열하지 마세요."
```

**점수 근거: 78점** — 톤과 내용은 75점에서 개선됐지만, 구조 라벨 노출이 치명적. 이대로 복붙하면 "제목\n브릿츠메디..."로 나가므로 실무 사용 불가. -12점 감점.
