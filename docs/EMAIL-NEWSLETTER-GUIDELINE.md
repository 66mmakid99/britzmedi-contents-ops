# BRITZMEDI 이메일 뉴스레터 가이드라인

> **적용 대상:** Content Factory `src/constants/prompts.js` → `case 'email'`  
> **목적:** 보도자료 → 이메일 뉴스레터 변환 + Derm Future Radar 큐레이션 뉴스레터  
> **핵심 원칙:** 열게 하고(open) → 훑게 하고(scan) → 클릭·회신하게(CTA)

---

## 1. 기본 원칙

### 독자 페르소나
- **주 독자:** 피부과/에스테틱 원장님 (의사)
- **특성:** 바쁘고, 새 장비/기술 트렌드에 관심, 직접적 이익(매출·환자 만족)에 민감, 업계 동향을 빠르게 파악하고 싶어함
- **읽는 환경:** 모바일 우선, 진료 합간에 빠르게 훑어봄

### 톤앤매너
- 존칭 사용 (~습니다, ~하셨을 겁니다)
- 전문적이되 따뜻한 톤 — 딱딱한 보고서 ✕, 친근한 전문가 브리핑 ○
- "원장님" 호칭을 자연스럽게 2~3회 사용
- 업계 전문용어 적절히 활용 (EBD, 토로이덜 고주파 등)
- 교육 중심 톤 — 프로모션보다 가이드·인사이트·케이스 비중 높게

### 글자수
- **총 1,500~2,500자** (블록 구조로 체감 길이는 짧게)
- 한 블록당 3~5문장 제한

## 2. 전체 구조: 8파트

```
[제목] Subject line — 오픈율을 결정하는 첫 관문
[프리헤더] Preview text — 받은편지함에서 제목을 보완

① 오프닝 (2~3문장)
   인사 + 이번 호에서 얻을 것 목차형 미리보기

② 블록 1: 핵심 뉴스/트렌드 (소제목 + 3~5문장 + 더보기 링크)
③ 블록 2: 인사이트/논문 (소제목 + 3~5문장 + 더보기 링크)
④ 블록 3: 실무 팁/기술 (소제목 + 3~5문장 + 더보기 링크)
   ※ 블록 수는 콘텐츠에 따라 2~4개 유동적

⑤ Primary CTA (핵심 행동 1개)
⑥ 푸터 (발신자·회사 정보·구독 관리)
```

## 3. 파트별 상세 가이드

### [제목] Subject Line

**규칙:**
- **"베네핏 + 구체성"** 조합 필수
- **40자 내외** (모바일에서 잘리지 않는 길이)
- 숫자·시간성(이번 달, 이번 주) 사용이 효과적
- 스팸성 단어 금지 (!!!, 무료, 긴급, 대문자 남용)

**좋은 예:**
- "이번 달 피부과 트렌드 3가지 요약"
- "RA 담당자가 꼭 봐야 할 새 가이드 2개"
- "토르RF 해외 임상 결과, 원장님께 드리는 요약"
- "2026 동남아 에스테틱 시장, 5분 브리핑"

**나쁜 예:**
- "브릿츠메디 뉴스레터 제3호" (가치 제안 없음)
- "꼭 읽어주세요!!! 중요 공지!!!" (스팸 느낌)
- "안녕하세요, 브릿츠메디입니다" (인사를 제목에 넣음)

### [프리헤더] Preview Text

**규칙:**
- 제목에서 못 담은 **구체적 가치를 보완**하는 "두 번째 제목"
- 첫 문장을 그대로 노출시키지 말고, **전용 문구** 설정
- 시간·분량 약속이 효과적 ("5분 안에 훑어보기")

**예시:**
- 제목: "이번 달 피부과 트렌드 3가지 요약"
- 프리헤더: "AI·장비·인허가를 5분 안에 훑어보는 CMO/RA 전용 브리핑"

### ① 오프닝 (2~3문장)

**기존 강점 유지:** "원장님, 안녕하세요. 브릿츠메디입니다." 인사

**추가:** 목차형 가치 미리보기
- "이번 호에서는 ① ○○, ② ○○, ③ ○○을 정리했습니다."
- 바로 독자 가치로 들어가기 — 회사 이야기·자기 소개 길게 ✕

**예시:**
```
원장님, 안녕하세요. 브릿츠메디입니다.
이번 호에서는 ① 태국 시장 진출 소식과 원장님 클리닉에 주는 의미,
② FDA 승인 토로이덜 고주파 기술 요약, ③ 동남아 시장 전망을 정리했습니다.
5분이면 충분합니다.
```

### ②③④ 본문 블록 (각 블록 동일 구조)

**핵심 변경: 순차 나열 → 독립 블록 구조**

각 블록 구조:
```
[소제목] — 호기심 유발형 또는 가치 제안형
[본문] — 2~4문장 요약 (인버티드 피라미드: 중요한 것 먼저)
[더 보기] — 링크 또는 버튼
```

**소제목 규칙:**
- 질문형: "태국 피부과 시장, 왜 지금 주목받을까요?"
- 가치형: "원장님 클리닉에 주는 3가지 의미"
- 대괄호([ ]) 금지 → 자연어 소제목

**블록 안에 인사이트 녹이기 (기존 ③ 강점 유지):**
- 기존의 "원장님 관점 인사이트" 섹션을 없애지 않음
- 대신 각 블록의 요약 안에 **"이것이 원장님께 왜 의미 있는지"**를 자연스럽게 포함
- 보도자료에 없는 원장님 관점의 해석을 반드시 추가

**예시 — 블록 1:**
```
📌 태국 독점유통 계약, 원장님 클리닉에 주는 의미

브릿츠메디가 태국 더마 솔루션스와 토르RF 독점유통 계약을 체결했습니다.
3년간 연 300대 규모입니다.

이 소식이 국내 원장님들께도 의미 있는 이유가 있습니다.
해외 시장에서의 기술력 검증은 곧 국내 장비 신뢰도 향상으로 이어집니다.
글로벌 임상 현장에서 축적되는 데이터는 원장님 클리닉의 시술 프로토콜에도
참고할 수 있는 근거가 됩니다.

👉 자세한 계약 내용 보기
```

**예시 — 블록 2:**
```
📌 FDA 승인 토로이덜 고주파, 어떤 점이 다를까요?

토르RF의 핵심 기술인 토로이덜 고주파(Toroidal RF)는
기존 모노폴라·바이폴라와 달리 에너지 분포가 균일합니다.
시술 효과의 일관성이 높아 다운타임 예측이 수월해지고,
환자 만족도 관리에도 유리하다는 임상 보고가 늘고 있습니다.

👉 기술 상세 보기
```

**블록 수 가이드:**
- 보도자료 변환: 2개 블록 (뉴스 + 기술/의미)
- Derm Future Radar: 3개 블록 (트렌드 + 논문 + 실무팁)
- 한 블록당 하나의 메시지, 하나의 행동만

### ⑤ CTA 설계

**이원화 구조:**

| 위치 | 종류 | 용도 |
|------|------|------|
| 블록 안 | Secondary CTA | "자세히 보기", "논문 전문 읽기" (텍스트 링크) |
| 하단 | Primary CTA | 이번 호의 가장 중요한 행동 1개 |

**Primary CTA 예시 (상황별):**
- 리드 육성: "이런 인사이트를 팀에 정기 브리핑으로 받고 싶으시면, 이 이메일에 회신해 주세요."
- 상담 유도: "토르RF에 대해 더 자세히 알고 싶으시다면 담당자에게 문의해주세요."
- 이벤트: "웨비나 자리 예약하기"
- 피드백: "이번 뉴스레터가 도움이 되셨나요? 한 줄 회신 주시면 다음 호에 반영하겠습니다."

**CTA 문구 규칙:**
- 간결하고 행동 지향적: "전체 보기", "데모 요청", "회신하기"
- 모호한 CTA 금지: "자세히 보기 → britzmedi.co.kr" (어디로 가는지 불명확)

## 4. 포맷·가독성 규칙

### 스캔 가능성 (훑어보기)
- 한 문단 2~4문장 (길면 분리)
- 각 블록에 반드시 소제목
- 핵심 수치·키워드는 **강조** 가능 (과도한 볼드 금지)
- 블록 사이 구분선(—) 또는 여백

### 모바일 최적화
- 단일 컬럼 기준으로 작성
- 짧은 문단, 넉넉한 줄바꿈
- CTA는 버튼형 (모바일에서 탭 가능한 크기)
- 한 줄에 50자 이내 권장

### 읽기 쉬운 언어 (건강 문해력)
- 전문용어 사용 가능하되, 괄호로 부연
  - 예: "토로이덜 고주파(Toroidal RF, 도넛 형태로 에너지를 전달하는 방식)"
- 단계별 안내가 필요하면 번호 사용
- 약어 첫 등장 시 풀네임 병기

## 5. 금지사항

| 금지 항목 | 이유 |
|----------|------|
| 보도자료 문장 그대로 복사 | 뉴스레터 톤이 아님 |

| "뷰티 디바이스", "K-뷰티", "미용기기" | 브릿츠메디 용어 정책 |

## 8. prompts.js 통합 가이드

### case 'email' 프롬프트에 반영할 핵심

```
기존 유지 항목:
✅ 피부과/에스테틱 원장님 대상 뉴스레터 작가 역할
✅ 존칭(~습니다) 사용, 따뜻하고 전문적인 톤
✅ "원장님" 호칭 자연스럽게 2~3회
✅ 보도자료 그대로 복사 금지 → 원장님 관점 재해석
✅ 전문용어 적절히 활용하되 설명적으로
✅ "뷰티 디바이스", "K-뷰티", "미용기기" 금지
✅ 1,500~2,500자

추가할 항목:
➕ 제목(Subject line) 별도 생성: 베네핏+구체성, 40자 내외
➕ 프리헤더(Preview text) 별도 생성: 제목 보완 문구
➕ 오프닝: "이번 호에서는 ①②③" 목차형 미리보기
➕ 본문: 2~3개 독립 블록 (소제목 → 요약 → 더보기 링크)
➕ CTA 이원화: 블록 내 Secondary + 하단 Primary
```
