# 브릿츠메디 네이버 블로그 콘텐츠 가이드라인 (2025–2026)

> 기존 prompts.js 강점 + 2025–2026 네이버 SEO 베스트 프랙티스 통합

---

## 1. 블로그 정체성 & 독자 페르소나

### 블로그 정체성
- **메인 주제**: 피부과·에스테틱 의료기기 (RF, HIFU, 레이저 등)
- **확장 주제**: RA/인허가, 글로벌 세일즈, 클리닉 경영, 시술 트렌드
- 하나의 축(메인 주제) 안에서 질문을 쪼개어 여러 글로 확장
- 관련 없는 주제 혼합 금지 → 네이버 카테고리·주제 매칭 평가에 중요

### 독자 페르소나
| 구분 | 내용 |
|------|------|
| 1차 독자 | 피부과·에스테틱 클리닉 원장, 장비 도입 의사결정자 |
| 2차 독자 | 의료기기 제조사 RA·수출 담당자, 유통 파트너 |
| 관심사 | 장비 선택 기준, 시술 효과·안전성, 인허가 절차, 해외시장 동향 |
| 검색 패턴 | 질문형 ("RF 리프팅 장비 비교", "의료기기 해외인허가 순서") |

## 2. 제목 작성 규칙

### 기본 규칙
- **글자수**: 30~40자 (한글 기준)
- **핵심 키워드**: 제목 앞부분에 배치
- **질문형 우선**: 실제 검색 의도를 그대로 반영한 질문형 제목 권장

### 제목 유형별 템플릿
| 유형 | 예시 |
|------|------|
| 질문형 (최우선) | "피부과 RF 장비, 어떤 기준으로 골라야 하나요?" |
| How-to형 | "의료기기 해외 인허가, 처음부터 끝까지 5단계로 정리" |
| 리스트형 | "피부과 원장이 RF 장비 선택 전 확인해야 할 3가지" |
| 비교형 | "모노폴라 vs 바이폴라 RF, 무엇이 다를까?" |

### 금지사항
- 과도한 클릭베이트 (충격!, 대박!, 역대급)
- 특수문자 남용 (★, ※, ♥ 등)
- 보도자료 제목 그대로 복사
- "뷰티 디바이스", "K-뷰티", "미용기기" 사용 금지 (자사 용어 정책)

## 3. 본문 3단 구조

### ① 서론 — 첫 200자 (네이버가 가장 중요하게 보는 영역)

**반드시 포함할 3요소:**
1. **이 글이 답하려는 질문/문제** 명시
2. **대상 독자** 한 줄 (누구를 위한 글인지)
3. **핵심 포인트 2~3개** 미리보기

**핵심 키워드를 첫 200자 안에 자연스럽게 배치**

**서론 템플릿:**
```
"이 글에서는 '[질문]'에 대해 정리합니다.
[대상 독자]를 위해, ① [포인트1], ② [포인트2], ③ [포인트3] 순서로 설명하겠습니다."
```

**예시:**
```
"이 글에서는 '국내 피부과용 RF 장비를 수출할 때 RA 준비는 어떤 순서로 진행해야 하나요?'라는 질문에 답합니다. 해외 인허가를 처음 시도하는 국내 제조사·수출 담당자를 대상으로, ① 시장별 분류·등급 확인, ② 필수 기술문서, ③ 일정·비용 설계 순으로 정리했습니다."
```

---

### ② 본론 — 질문형 소제목 + 답변 패턴

#### 소제목 규칙
- **질문형 소제목**을 기본으로 사용 (실제 사용자 검색 의도 반영)
- 소제목 3~5개로 구조화

**소제목 템플릿:**
| 패턴 | 예시 |
|------|------|
| ~은 무엇인가요? | "모노폴라 RF란 무엇인가요?" |
| ~은 어떻게 하나요? | "FDA 510(k) 신청은 어떻게 진행하나요?" |
| 왜 ~이 중요한가요? | "왜 RF 주파수 대역 선택이 중요한가요?" |
| ~할 때 주의할 점은? | "RF 장비 수출 시 주의할 점은 무엇인가요?" |
| ~ vs ~ 비교 | "모노폴라 vs 바이폴라, 임상 결과 차이는?" |

#### 각 소제목 아래 답변 구조 (4단 패턴)
```
[한 줄 직접 답] — 핵심 결론을 먼저 제시
[정의/원리] — 배경 설명 2~3문장
[사례/숫자/경험] — 구체적 데이터, 실제 사례, 프로젝트 경험
[FAQ/자주 틀리는 포인트] — "많이들 오해하시는 부분인데요..." 1~2개
```

**예시:**
```
## FDA 510(k) 신청은 어떻게 진행하나요?

한마디로 요약하면, 기존 유사 장비(Predicate Device)를 찾는 것이 첫 단계입니다.

510(k)는 미국 FDA에서 Class II 의료기기에 요구하는 시판 전 신고 절차인데요.
핵심은 "이미 허가된 유사 장비와 동등하다"는 것을 입증하는 과정입니다.

실제로 국내 RF 장비 제조사의 경우, Predicate 선정부터 최종 클리어런스까지
평균 8~12개월이 소요됩니다. 비용은 약 $12,000~$15,000(FDA 수수료 기준)입니다.

자주 오해하시는 부분이 있는데요 — 510(k)는 '허가'가 아니라 '신고(Clearance)'입니다.
PMA(시판 전 승인)와는 완전히 다른 절차이므로 혼동에 주의하세요.
```

### ③ 결론 — 마지막 200자

**반드시 포함할 3요소:**

```
관련해서 더 궁금한 점이 있으시면 댓글로 남겨 주세요.
▶ 관련 글: [내부 링크 제목]"
```

## 4. 키워드 전략

### 키워드 구분
| 구분 | 개수 | 역할 | 예시 |
|------|------|------|------|
| 핵심 키워드 | 2~3개 | 제목·서론·결론 필수 배치 | "RF 장비", "피부과 의료기기" |
| 보조 키워드 | 5~7개 | 본론에 자연스럽게 분포 | "모노폴라", "바이폴라", "콜라겐 리모델링" |
| 총합 | 10개 내외 | 글 전체에 자연스럽게 | — |

### 배치 규칙
- 핵심 키워드: 제목 + 서론 첫 200자 + 결론 마지막 200자에 반드시 포함
- 보조 키워드: 각 소제목·본론에 1~2개씩 자연 배치
- **스터핑 금지**: 같은 키워드를 한 단락에 3회 이상 반복하지 않음

### 카테고리 매칭
- 블로그 카테고리와 글 주제가 일치해야 함 (네이버 2026 평가 핵심)
- "피부과 장비" 카테고리에 "맛집 후기"가 섞이면 전체 블로그 평가 하락

## 5. 내부 링크 & 출처

### 내부 링크 (필수)
- 결론부에 관련 글 1~2개 링크
- 본론 중에도 "이 부분은 [이전 글 제목]에서 자세히 다뤘는데요" 형태로 자연 삽입
- 동일 주제군 글끼리 촘촘하게 연결 → 네이버 "전문성·문맥" 평가에 유리

### 외부 출처 (권장)
- 공공기관, 논문, 공식 통계 등 신뢰도 높은 출처를 인용할 때 링크 연결
- "공신력 신호"로 작용
- 예시: FDA 공식 사이트, MFDS(식약처) 공지, 논문 DOI 링크
- 형식: "FDA 가이던스에 따르면(출처: fda.gov/…), …"

## 7. 가독성 규격

### 문장·단락 기준
| 항목 | 기준 |
|------|------|
| 한 문장 | 30~50자 (한글 기준) |
| 한 단락 | 300~500자 (3~5문장) |
| 전체 글 | 2,500~3,500자 (최대 4,000자) |
| 소제목 수 | 3~5개 |

### 톤앤매너
- **친근하되 전문적인 블로그 톤**
- 사용: "~인데요", "~했습니다", "~하시면 됩니다"
- **금지**: "~했다", "~밝혔다", "~전했다" (보도문체)
- 전문 용어 사용 시 괄호 설명 병기: "Predicate Device(기존 유사 인허가 장비)"

### 자사 표기 규칙
| 항목 | 규칙 |
|------|------|
| 제품명 | 토르RF, 루미노웨이브, 뉴채, 울블랑 (정확히 표기) |
| 영문 | 첫 등장: "국문(영문)", 이후 국문만 |
| 금지 용어 | "뷰티 디바이스", "K-뷰티", "미용기기" |
| 의료법 금지어 | 최고, 최초, 유일, 획기적, 혁신적, 완치, 기적 등 |

## 8. 금지사항 종합

1. ❌ 보도자료 본문 복사·붙여넣기
2. ❌ 보도문체 ("~했다", "~밝혔다")
3. ❌ 키워드 스터핑 (같은 키워드 과도 반복)
4. ❌ 의료법 금지어 (최고, 최초, 유일, 획기적, 완치 등)
5. ❌ "뷰티 디바이스", "K-뷰티", "미용기기"
6. ❌ 관련 없는 주제 혼합 (카테고리 불일치)
7. ❌ 장식용 이미지 남용 (내용과 무관한 스톡 사진)
8. ❌ 출처 없는 통계·수치 인용
9. ❌ 클릭베이트 제목 (충격!, 대박!, 역대급)
10. ❌ 새로운 팩트 창작 (원본에 없는 내용 추가)

## 12. prompts.js 통합 가이드

### 기존 → 신규 변경 요약

| 기존 prompts.js | 변경 내용 |
|----------------|----------|
| "SEO 키워드가 포함된 매력적인 블로그 제목" | → 질문형/How-to형, 30~40자, 핵심 키워드 앞배치 |
| (서론 규칙 없음) | → 첫 200자: 질문 → 대상독자 → 핵심포인트 2~3개 |
| "3~5개의 H2 소제목" | → **질문형** 소제목 3~5개 |
| "각 소제목 아래 2~3개 문단" | → **4단 답변 패턴**: 한줄답 → 정의 → 사례 → FAQ |
| "SEO 키워드: 5~8개 추출" | → 핵심 2~3개 + 보조 5~7개 구분, 배치 규칙 명시 |
| "[IMAGE: 설명]" | → "[IMAGE: alt 텍스트 20~50자 + 키워드]", 10~15장 |
| "CTA: 마지막에 자연스러운 행동 유도" | → 결론 200자: 요약 3~5줄 + 키워드 재언급 + CTA + 내부링크 |
| (없음) | → 내부 링크 삽입 (결론 + 본론 중 자연 삽입) |
| (없음) | → 출처·공신력 링크 1개 이상 |
| (없음) | → 문장 30~50자, 단락 300~500자, 전체 2,500~3,500자 |
| "1,500~2,500자" | → 2,500~3,500자 (상향) |

### prompts.js case 'naver-blog' 반영 시 핵심 포인트

1. **서론 구조** 추가: 첫 200자에 질문·대상·포인트 미리보기
2. **질문형 소제목** 규칙 명시 + 템플릿 패턴 제공
3. **4단 답변 패턴** 추가: 한줄답 → 정의 → 사례 → FAQ
4. **결론 구조** 추가: 요약·키워드 재언급·CTA·내부링크
5. **키워드 이원화**: 핵심 2~3개 (제목·서론·결론) + 보조 5~7개 (본론)
6. **이미지 강화**: alt 텍스트 규칙, 장수 기준, 키워드 포함
7. **가독성 규격**: 문장·단락·전체 글자수 기준 명시
8. **내부 링크 + 출처** 삽입 규칙 추가
9. **출력 포맷** 확장: 서론/본론/결론 구분 + 핵심·보조 키워드 분리
10. **분량 상향**: 1,500~2,500자 → 2,500~3,500자
