# BRITZMEDI 보도자료 작성 지침 v3

> **이 문서는 AI 보도자료 생성의 절대 지침이다.**
> 생성, 검수, 자동수정, 인용문 제안 모든 단계에서 이 문서를 반드시 준수한다.
> 이 문서의 규칙과 다른 지시가 충돌하면, 이 문서가 우선한다.
>
> 최종 수정: 2026-02-16

---

## 1. 본문 구조: 5단 역피라미드

모든 보도자료 본문은 반드시 아래 5단 구조를 따른다.
팩트 나열이 아닌, 논리적 흐름을 가진 하나의 이야기를 만든다.

### ■ 1단락: 리드 (WHAT + WHO + WHEN + WHERE)

- 핵심 뉴스를 한 문단(2~3문장)으로 완결 요약
- 5W 중 가장 중요한 것을 **첫 문장**에 배치
- 소스의 **모든 핵심 숫자**(계약 기간, 수량, 날짜)를 리드에 반드시 포함
- 리드만 읽어도 무슨 뉴스인지 알 수 있어야 한다

```
✅ 좋은 예:
"메디컬 에스테틱 디바이스 전문기업 브릿츠메디(대표 이신재)가
태국 방콕 소재 더마 솔루션스(Derma Solutions)와
3년간 연 300대 규모의 토르RF(TORR RF) 독점유통 계약을 체결했다."

❌ 나쁜 예:
"브릿츠메디가 해외 파트너사와 계약을 체결했다."
→ 구체성 없음. 기간, 규모, 상대방 누락.
```

### ■ 2단락: WHY — 배경과 맥락

- 이 뉴스가 **왜** 나왔는지 배경 설명
- 회사 전략, 시장 상황, 업계 흐름과 연결
- 지식베이스의 회사/제품 정보를 활용하여 맥락 제공
- **단, 보일러플레이트(회사 설립연도, 소재지, 일반 사업 소개)는 쓰지 않는다** → 그건 별도 [회사 소개] 섹션의 몫

```
✅ 좋은 예:
"이번 계약은 브릿츠메디의 동남아시아 시장 확대 전략의 일환이다.
태국은 아세안 에스테틱 의료 시장의 핵심 거점으로,
연간 시장 규모가 지속적으로 성장하고 있는 지역이다."

❌ 나쁜 예:
"브릿츠메디는 2017년 설립된 메디컬 에스테틱 디바이스 전문기업으로..."
→ 보일러플레이트 중복. 본문에 넣지 않는다.
```

### ■ 3단락: HOW — 구체적 내용

- 계약 조건, 제품 스펙, 행사 세부 내용 등 구체적 팩트
- 소스의 숫자 팩트가 이 단락에 집중 배치
- 기술적 내용은 비전문가도 이해할 수 있는 수준으로 풀어서 설명
- 파트너사 소개가 필요하면 이 단락에서 간략히

```
✅ 좋은 예:
"4월 15일 방콕 본사에서 진행된 계약식을 통해 양사는
올해 하반기부터 납품을 시작하기로 합의했다.
더마 솔루션스는 태국 내 주요 피부과 네트워크를 보유한
전문 유통 기업으로, 다년간의 현지 유통 경험을 갖추고 있다."
```

### ■ 4단락: SO WHAT + 인용문 — 의미와 관계자 코멘트

- 이 뉴스의 **산업적/비즈니스적 의미**를 먼저 서술
- 이어서 **관계자 인용문**을 자연스럽게 배치
- 인용문은 이 단락 안에 녹여서 쓴다. 별도 섹션으로 분리하지 않는다
- 인용문 뒤에 자연스러운 부연 또는 전환 문장 추가

```
✅ 인용문 배치 예:
"이번 파트너십을 통해 브릿츠메디는 동남아 시장에서의 입지를
본격적으로 확대할 것으로 기대된다.
이신재 대표이사는 "태국은 동남아 에스테틱 시장의 핵심 거점으로,
이번 파트너십을 시작으로 아세안 전역에서 토르RF의 가치를
입증해 나갈 것"이라고 밝혔다."

❌ 나쁜 예:
"[대표 인용문]
이신재 대표이사: '...'
"
→ 본문 밖 별도 섹션 금지. 반드시 4단락 안에 녹인다.
```

### ■ 5단락: 마무리 — 향후 계획

- 향후 일정, 추가 계획, 전망으로 마무리
- 자연스러운 종결감이 있어야 한다
- 기술 스펙 나열이나 숫자로 끝나면 안 된다

```
✅ 좋은 예:
"브릿츠메디는 올해 하반기 태국 납품을 시작으로
동남아시아 주요 국가로의 추가 파트너십 확대를 계획하고 있다."

❌ 나쁜 예:
"콜라겐이 65.8%에서 81.2%로 증가한다."
→ 기술 스펙으로 끝남. 마무리 아님.
```

## 2. 단락 연결 규칙

- 각 단락의 첫 문장은 이전 단락과 **자연스럽게** 이어져야 한다
- 맥락으로 연결한다:
  - "이번 계약의 배경에는..."
  - "구체적으로 살펴보면..."
  - "이를 통해..."
- **기계적 접속사 남발 금지** (아래 단어 2회 이상 사용 금지):
  - "한편", "또한", "아울러", "이와 함께", "이에 따라"
- 단락 간 논리 흐름: **무엇 → 왜 → 어떻게 → 그래서 → 앞으로**

## 3. 숫자 팩트 규칙 (CRITICAL — 절대 누락 금지)

### 3-1. 숫자 완전성 검증 프로세스

```
1단계: 소스에서 모든 숫자 표현을 추출
       (기간, 수량, 금액, 날짜, 퍼센트, 연도, 면적, 인원)
2단계: 추출한 숫자 목록을 체크리스트로 작성
3단계: 본문 작성 완료 후 체크리스트 1:1 대조
4단계: 누락된 숫자가 있으면 적절한 단락에 반드시 삽입
```

### 3-2. 숫자 배치 원칙

| 숫자 유형 | 배치 위치 |
|----------|----------|
| 계약 기간, 수량, 핵심 규모 | **1단락(리드)** — 반드시 |
| 구체적 날짜, 장소, 세부 조건 | **3단락(HOW)** |
| 기술 스펙, 성능 수치 | **3단락(HOW)** |
| 향후 일정, 시기 | **5단락(마무리)** |

### 3-3. 쌍(pair) 규칙

기간과 수량이 쌍으로 나오면 반드시 **함께** 언급한다:
- "3년 계약, 연 300대" → "3년간 연 300대 규모"
- "3년"만 쓰거나 "300대"만 쓰면 **불완전** → 검수에서 critical

## 4. 인용문 규칙

### 4-1. 발화자 선택 기준

| 카테고리 | 기본 발화자 | 이유 |
|----------|-----------|------|
| 파트너십·계약 | CEO 대표이사 | 경영 전략 의사결정 |
| 실적·투자 | CEO 또는 CFO | 재무 성과 관련 |
| 신제품·기술·인증 | CTO 기술이사 | 기술 전문성 |
| 전시회·마케팅 | CMO 마케팅총괄 | 시장 전략 |
| 인사·수상 | CEO 대표이사 | 대표 발언 적합 |

### 4-2. 인용문 작성 원칙

- **분량**: 1~3문장
- **위치**: 반드시 4단락(SO WHAT) 안에 자연스럽게 녹인다
- **형식**: `{이름} {직책}은(는) "인용문"이라고(이라며) 밝혔다(말했다/강조했다).`
- **톤**: 구어체 금지. 보도문체 격식 유지.
- **내용**: 해당 뉴스의 전략적 의미, 비전, 기대효과를 담는다.
- **금지**: 소스에 없는 구체적 숫자나 팩트 날조. 의료법 금지어.

### 4-3. 인용문 3안(A/B/C)

AI는 항상 3가지 인용문 안을 제안한다:

| 안 | 톤 | 초점 |
|----|-----|------|
| A안 | 전략적 | 비즈니스 전략, 시장 확대, 비전 |
| B안 | 기술적 | 기술 차별화, 제품 전문성, 혁신 |
| C안 | 관계형 | 파트너 협력, 고객 가치, 신뢰 구축 |

### 4-4. [QUOTE_PLACEHOLDER] 사용 금지

- 본문에 `[QUOTE_PLACEHOLDER]`, `[인용문]`, `[대표 인용문]` 같은 플레이스홀더를 넣지 않는다.
- AI가 생성 시 기본 인용문(A안)을 본문 4단락에 바로 포함시킨다.
- 사용자가 B안 또는 C안을 선택하면, 4단락의 인용문 부분만 교체한다.

## 5. 보도문체 규칙

### 5-1. 문장 톤

- 종결어미: "~했다", "~밝혔다", "~전했다", "~계획이다" (보도문체)
- 금지: "~인데요", "~했습니다", "~거든요" (블로그/구어체)
- 금지: "~할 것으로 보인다" (추측형) → 대신 "~할 계획이다", "~예정이다"

### 5-2. 용어 규칙

| 사용 금지 | 대체 표현 |
|----------|----------|
| 행사 기간 | 학술대회 기간 / 학회 일정 중 / 전시 기간 |
| 행사 (단독) | 학술대회 / 학회 / 전시회 / 콘퍼런스 (정확한 명칭) |
| 이벤트 | 행사 / 프로그램 / 세션 |
| 엄청난 / 대단한 | (삭제하거나 구체적 수치로 대체) |

### 5-3. 학술회·전시회 관련 용어

- "학회 참가" ✅ / "행사 참가" ❌
- "학술대회 기간 중" ✅ / "행사 기간 중" ❌
- "라이브 시연" 또는 "라이브세션" ✅ / "라이브 쇼" ❌
- "부스 전시" ✅ / "부스 운영" ✅ / "부스 오픈" ❌

## 6. 영문 표기 규칙

- **모든 영문**(자사 + 파트너사 + 기관 + 행사 + 기술명): 첫 등장 시 `국문(영문)`, 이후 국문만
- 예: "더마 솔루션스(Derma Solutions)" → 이후 "더마 솔루션스"
- 예: "미국 식품의약국(FDA)" → 이후 "FDA" (약어는 예외적으로 영문 유지 가능)
- 자사 제품명: 토르RF(TORR RF), 루미노웨이브(LuminoWave), 뉴채(NEWCHAE), 울블랑(ULBLANC)
- 기술명: 토로이달(TOROIDAL)

## 7. 의료법 준수

### 7-1. 절대 금지어 (사용 시 검수에서 critical)

```
극대화, 최소화, 최고, 최초(무근거), 유일, 완벽한, 획기적인,
혁명적인, 혁신적인, 완치, 100%, 확실한 효과, 부작용 없음,
안전 보장, 비교 광고(타사 제품명 직접 언급), 효능 단정
```

### 7-2. 허용 대체 표현

| 금지 | 대체 |
|------|------|
| 극대화 | 개선 / 촉진 / 향상 |
| 최소화 | 감소 / 저감 |
| 완벽한 | 우수한 / 높은 수준의 |
| 효과 확실 | 효과가 기대되는 |
| 획기적인 | 차별화된 / 독자적인 |
| 부작용 없음 | 부작용 가능성을 낮춘 |

## 8. 팩트 사용 규칙 — 할루시네이션 방지

### 8-1. 3단계 원칙

```
1순위: 소스 (사용자 입력 팩트) → 뼈대 → 반드시 사용
2순위: 지식베이스 (등록된 정보) → 살 → 맥락 보강에 활용
3순위: 없으면 → 절대 만들지 않음 → [입력 필요] 플레이스홀더
```

### 8-2. 유추 허용 범위

**허용:**
- 유통 계약 상대방 → "유통을 담당하는 파트너사" 수준 설명
- 특정 국가 계약 → 해당 지역권 언급 (태국 → 동남아시아)
- 계약 후 → "추가 파트너십 확대 계획" 같은 일반적 향후 전망
- 특정 학회 참가 → 해당 학회의 일반적 성격·규모 설명

**절대 금지:**
- 구체적 숫자 날조 (참석자 수, 미팅 건수, 매출액)
- 구체적 회사 상세 날조 (설립연도, 직원수, 매출 규모)
- 현장 반응, 참가자 코멘트 날조
- 출처 없는 시장 데이터, 시장 규모 수치

### 8-3. [입력 필요] 플레이스홀더

소스와 지식베이스 모두에 없는 정보가 필요할 때:
```
[입력 필요: 파트너사 설립연도]
[입력 필요: 계약 세부 조건]
```
절대 추측으로 채우지 않는다.

## 9. 본문에 포함하지 않는 것

아래 항목은 본문 밖에서 별도로 관리한다. **본문 안에 넣지 않는다.**

| 항목 | 위치 |
|------|------|
| 회사 소개 (보일러플레이트) | [회사 소개] 별도 섹션 |
| 태그 / 해시태그 | 보도자료에는 포함하지 않음 |
| 사진 가이드 | 작성자 참고용 (아웃풋 제외) |
| 첨부파일 가이드 | 작성자 참고용 (아웃풋 제외) |
| 뉴스와이어 입력 필드 | 작성자 참고용 (아웃풋 제외) |

**"작성자 참고용"**이란:
- 화면 UI에서는 보이되, 연회색 배경 + "📌 작성자 참고용" 라벨로 구분
- Word 다운로드, PDF, 클립보드 복사에는 **절대 포함하지 않음**

## 10. 카테고리별 맥락 가이드

### 전시회·학회 참가

```
리드: 어떤 학회에 참가해서 무엇을 전시/발표하는지
배경: 해당 학회의 위상, 참가 목적, 전략적 의미
구체: 부스 위치, 전시 제품, 라이브세션, 발표 내용
의미: 시장 반응 기대, 기술력 입증 기회 + 관계자 코멘트
마무리: 학회 일정, 향후 참가 계획
```

### 파트너십·계약

```
리드: 누구와 무슨 계약을 어떤 규모로 체결했는지
배경: 해당 시장의 전략적 중요성, 진출 배경
구체: 계약 조건 상세, 파트너사 역량, 일정
의미: 시장 확대 효과, 매출 기대 + 관계자 코멘트
마무리: 납품 시작 시기, 추가 확장 계획
```

### 신제품·신기술 출시

```
리드: 무슨 제품이 언제 출시되는지 + 핵심 차별점 한 줄
배경: 개발 배경, 기존 제품 한계, 시장 니즈
구체: 기술 스펙 (이해하기 쉽게), 기존 대비 개선점
의미: 시장 영향, 적용 분야 확대 + 관계자 코멘트
마무리: 출시 일정, 인증 계획, 프로모션
```

### 인증·임상 성과

```
리드: 어떤 인증/승인을 받았는지
배경: 해당 인증의 의미, 왜 중요한지
구체: 인증 과정, 심사 기준, 임상 결과
의미: 시장 진출 효과, 경쟁력 강화 + 관계자 코멘트
마무리: 다음 인증 계획, 시장 진출 일정
```
