# BRITZMEDI Content Factory — LinkedIn 콘텐츠 가이드라인

> **버전:** 1.0  
> **최종 업데이트:** 2026.02.17  
> **적용 대상:** prompts.js → case 'linkedin' 프롬프트 및 LinkedIn 콘텐츠 전체

---

## 1. 기본 원칙

### 1-1. 한 포스트 = 하나의 메시지
- 여러 주제를 섞지 않는다.
- "하나의 통찰" 또는 "하나의 스토리" 또는 "하나의 노하우"에 집중한다.

### 1-2. 네이티브 콘텐츠 우선
- 외부 링크(회사 블로그, 뉴스 URL 등)는 **본문이 아니라 첫 번째 댓글**에 배치한다.
- 본문에 외부 링크를 넣으면 알고리즘이 도달을 줄인다.
- 첫 댓글에 링크를 넣을 경우, 본문 마지막에 "링크는 첫 댓글에 남겨둡니다" 한 줄을 추가한다.

### 1-3. 대화 유도형 글쓰기
- "정보 전달"이 아니라 "대화거리 제공"을 목표로 설계한다.
- 모든 것을 다 설명하지 않고, **댓글에서 이어질 여지**를 남겨둔다.

### 1-4. 전문성 + 개인 경험
- 순수 이론·홍보 글보다, **실제 경험·실패·인사이트가 결합된 글**이 알고리즘과 사람 모두에게 유리하다.
- BRITZMEDI 맥락: 의료기기 수출, RA, 피부과 세일즈, 해외 파트너십 경험을 녹인다.

## 2. 글 구조: Hook → Body → CTA (3단 구성)

### 2-1. Hook (첫 1~2줄) — 스크롤을 멈추게

**역할:** 호기심·공감·가벼운 논쟁성을 만들어서 "더 보기" 클릭을 유도한다.

**유형 3가지:**

| 유형 | 설명 | BRITZMEDI 예시 |
|------|------|---------------|
| 놀라운 주장형 | 반직관적이거나 강한 주장으로 시작 | "좋은 제품만 있으면 해외 시장에서 팔린다? 3년 수출 경험으로 확인한 건, 정반대였습니다." |
| 갈등·문제 제기형 | 공통 실패 패턴이나 갈등 상황 제시 | "의료기기 해외 영업에서 가장 많이 하는 실수 — 제품 스펙부터 설명하는 것입니다." |
| 정답·치트키 선언형 | 실용적 해결책이 있다고 선언 | "피부과 장비 해외 세일즈, 파트너 선정 기준 딱 3가지만 보면 됩니다." |

**주의사항:**
- 너무 자극적인 클릭베이트는 알고리즘이 저품질로 분류할 수 있음
- Hook은 반드시 **한국어 1~2줄**(영문 서브라인 포함 시 3줄까지)로 제한

### 2-2. Body — 약속한 가치를 전달

**역할:** Hook에서 던진 질문·갈등·약속을 실제 정보·사례·리스트로 해소한다.

**작성 규칙:**

| 규칙 | 설명 |
|------|------|
| 1문단 = 1아이디어 | 한 문단에 여러 생각을 넣지 않는다 |
| 줄바꿈 자주 | 모바일에서 스캔하기 쉽게, 2~3줄마다 공백 라인 |
| 리스트/불릿 활용 | "3가지 포인트", "실패 이유 4가지" 등 구조화 |
| 나만의 디테일 | 자신의 경험·사례·숫자·미니 스토리 포함 |
| 대화 여지 남기기 | 모든 걸 다 설명하지 않고 댓글 유도 여지 확보 |

**BRITZMEDI 특화 Body 작성 포인트:**
- 기술 스펙을 나열하지 말고, "이 기술이 현지 시장에서 왜 먹히는지" 맥락을 연결
- 보도자료 문장을 그대로 가져오지 말고, 경험 기반 인사이트로 변환
- 숫자·데이터는 보도자료에 명시된 것만 사용 (AI가 자의 계산 금지)

### 2-3. CTA — 독자가 할 행동을 명확히 지시

**역할:** 댓글·팔로우·프로필 방문·DM 등 "다음 행동"을 구체적으로 요청한다.

**CTA 유형별 예시:**

| 유형 | 예시 |
|------|------|
| 경험 공유 요청 | "비슷한 경험 있으신 분, 댓글로 공유해 주세요." |
| 팔로우 유도 | "의료기기 수출·피부과 비즈니스 인사이트를 계속 보고 싶다면 팔로우해 주세요." |
| 의견 질문 | "여러분의 회사는 해외 파트너 선정 시 어떤 기준을 가장 중요하게 보시나요?" |
| DM 유도 | "더 자세한 내용이 궁금하시면 편하게 DM 주세요." |
| 피드백 요청 | "이 관점에서 빠진 게 있다면 댓글로 알려주세요." |

**CTA가 없으면 알고리즘이 좋아하는 "대화량(댓글)"을 끌어내기 어려움 — 반드시 포함.**

## 3. 포맷·문장 레벨 최적화

### 3-1. 글자수
- 목표: **800~1,200자** (한국어 기준)
- 너무 짧으면 깊이가 없고, 너무 길면 완독률이 떨어짐
- 영문 서브라인 포함 시 약간 초과 허용

### 3-2. 가독성 규칙
- 첫 두 줄(Hook) 이후는 줄바꿈 자주, 공백 라인으로 블록 분리
- 이모지는 **구조를 표시하는 용도로만 최소 사용** (장식용 남발 금지)
- B2B 의료기기 맥락에서 🚀, 🔥 같은 이모지는 가볍게 보일 수 있으므로 주의
- 적합한 이모지: ✅, 📌, 💡, ➡️, 🔎 (정보 구조화 용도)

### 3-3. 톤
- **전문적이되 말투는 대화체** — "논문체·보도자료체"는 반응이 낮음
- "~입니다" 존댓말 기반이지만, 딱딱하지 않게
- 자기 자랑보다 **배운 점·실패·교훈 중심의 스토리** → 3배 이상 높은 참여 유도

### 3-4. 한영 혼용 규칙
- 비즈니스 용어는 영어 유지: FDA, RA, EBD, KFDA, CE marking 등
- 영문 서브라인은 글로벌 오디언스 스캔용으로 Hook 바로 아래 배치
- 해시태그는 **영어 8~10개** (LinkedIn 글로벌 특성상 영어 해시태그가 검색에 유리)

## 4. 알고리즘 대응 운영 규칙

### 4-1. 초기 60분이 승부
- 게시 후 첫 60분의 반응(좋아요·댓글·체류시간)이 2차·3차 확장 노출을 결정
- 게시 직후 팀원/관계자에게 공유하여 초기 반응 확보 고려

### 4-2. 대화형 콘텐츠 우대
- 질문, 의견 요청, 논쟁 가능한 주제가 알고리즘에서 우대
- CTA를 통해 댓글을 적극 유도

### 4-3. 일관성 보너스
- 특정 주제(피부과 트렌드, 의료기기 RA, 해외 수출)를 꾸준히 다루면
  알고리즘이 "해당 분야 전문가"로 인식하여 도달을 높여줌

### 4-4. 스팸 시그널 회피
| 하지 말 것 | 이유 |
|-----------|------|
| 본문에 외부 링크 삽입 | 도달 감소 |
| 해시태그 10개 초과 | 스팸으로 인식 |
| 템플릿 티 나는 클릭베이트 | 저품질 분류 |
| "좋아요·팔로우 부탁" 반복 | 셀프 프로모션 감점 |
| 같은 내용 반복 게시 | 중복 콘텐츠 패널티 |

## 5. BRITZMEDI 전용 템플릿 (3종)

### 템플릿 A: 문제 제기형

```
[Hook] 대부분의 피부과 장비 세일즈가 LinkedIn에서 실패하는 이유, 사실 간단합니다.

[Body]
이유 1: (불릿 + 실제 사례/숫자 1줄)
이유 2: (불릿 + 실제 사례/숫자 1줄)
이유 3: (불릿 + 실제 사례/숫자 1줄)

(여기서 인사이트/학습 포인트 1~2문단)

[CTA] 당신은 어떤 이유 때문에 LinkedIn 세일즈가 어렵다고 느끼나요? 댓글로 경험을 공유해 주세요.

[해시태그] #MedicalDevice #Dermatology #B2BSales ...
```

### 템플릿 B: 트렌드·논문 요약형

```
[Hook] 최근 피부과 AI 진단 논문을 읽고, 실제 비즈니스에 바로 쓸 수 있는 포인트만 뽑아봤습니다.

[Body]
📌 논문 요약 3줄 (대상, 핵심결과, 한 줄 해석)

💡 이게 우리 비즈니스에 의미하는 것:
포인트 1:
포인트 2:
포인트 3:

[CTA] 이런 논문 기반 인사이트를 정기적으로 보고 싶으면 팔로우, 더 깊게 보고 싶은 논문이 있으면 댓글로 알려주세요.

[해시태그] #Dermatology #AI #ClinicalResearch ...
```

### 템플릿 C: 체크리스트형

```
[Hook] 의료기기 RA 기반으로 피부과 장비를 포지셔닝할 때, 제가 항상 체크하는 5가지입니다.

[Body]
✅ 체크 1: (짧은 설명)
✅ 체크 2: (짧은 설명)
✅ 체크 3: (짧은 설명 — 의외의 관점)
✅ 체크 4: (짧은 설명)
✅ 체크 5: (짧은 설명 — 차별적 관점)

[CTA] 여기서 빠졌다고 느끼는 항목이 있나요? 본인 체크리스트를 댓글로 추가해 주세요.

[해시태그] #RegulatoryAffairs #MedicalDevice #MarketAccess ...
```

## 7. 품질 체크리스트 (자체 검수용)

콘텐츠 생성 후 아래 항목을 확인합니다:

| # | 체크 항목 | 기준 |
|---|---------|------|
| 1 | Hook이 "더 보기" 클릭을 유도하는가? | 첫 2줄 이내 호기심/공감/논쟁 |
| 2 | Body에 나만의 경험/사례/숫자가 있는가? | 보도자료 직역이 아닌 인사이트 |
| 3 | CTA가 구체적 행동을 요청하는가? | 댓글/팔로우/DM 중 1개 이상 |
| 4 | 외부 링크가 본문에 없는가? | 링크는 첫 댓글 안내 |
| 5 | 글자수 800~1,200자인가? | 영문 서브라인 포함 |
| 6 | 해시태그 8~10개 영어인가? | 과다 해시태그 금지 |
| 7 | 포맷이 모바일 가독성을 충족하는가? | 1문단 1아이디어, 줄바꿈 |
| 8 | 톤이 대화체인가? | 논문체/보도자료체 금지 |
| 9 | 보도자료 수치를 자의 계산하지 않았는가? | 명시된 숫자만 사용 |
| 10 | 한영 혼용이 자연스러운가? | 비즈니스 용어 영어, 나머지 한국어 |
